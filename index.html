HTTP Header Information Disclosure

Solusi : Disarankan untuk tidak menampilkan server tersebut atau bilamana server ingin ditampilkan sebainya rubah nama server. Melakukan remove pada header tersebut, yaitu header: - X-Powered-By: PHP/8.1.25 - Server: openresty, Apache/2.4.1 Debian, Nginx/1.18 (*Contoh Hosting Server)

Berikut ini contoh informasi yang diungkapkan di header Server saat mengakses situs web yang dihosting di Apache :
Server: Apache/2.4.51 Debian (*exapmle server)
Ini dengan jelas mengungkapkan teknologi server, nomor versi dan sistem operasi apa yang digunakan. Untuk menghapus informasi ini dari header respons, tambahkan baris berikut ke file konfigurasi Apache. Contohnya dengan mengubah file /etc/apache2/apache2.conf, namun file bisa berbeda tergantung pada platform dan konfigurasi setiap developer menyimpanya.
ServerTokens Prod
ServerSignature Off (bisa di add di server,http,location)
Setelah menyimpan file konfigurasi, diperlukan restart server.
service apache2 restart

Untuk Server Nginx berikut langkahnya
Server: Nginx/1.18.0 (*example server)
Untuk menghapus informasi ini dari header respons, tambahkan baris berikut ke file konfigurasi nginx. Contohnya mengubah file /etc/nginx/nginx.conf. (*Noted :File dir berbeda – beda, sesuaikan dengan dir yang dev gunakan.)
server_tokens off; (bisa di add di server,http,location)
Setelah itu simpan konfigurasi tersebut, lalu restart server.
service nginx restart

Host Header Injection

Solusi : Mengganti header Host dengan header server_name atau bisa dengan melakukan beberapa penyesuaian pada /etc/httpd/conf/httpd.conf bila mengunakan server apache sebagai berikut :

RewriteEngine On
RewriteCond %{HTTP_HOST} !^(.*\.)?google\.com$ [NC]
RewriteRule .* - [L,R=403]

Pastikan e iRule sudah diset untuk bisa melewati bigip. Aturan ini juga mengambil header host dari permintaan kemudian mencocokkannya dengan regex yang memutuskan apakah ia harus mengizinkan permintaan tersebut atau tidak.

Cookie without SameSite, Httponly dan Secure Attribute

Solusi: Setting cookie tersebut agar memiliki flag samesite dengan attributes yang dapat disesuaikan sendiri (bisa menggunakan lax, strict, atau none). Penambahan cookie bisa dilakukan dalam server yang digunakan contoh seperti nginx atau apache tergantung pemakaian. Contoh penambahan bisa seperti berikut :
server {
location /wait {
proxy_cookie_path / "/; secure; httponly; samesite=lax atau strict";
}
}
